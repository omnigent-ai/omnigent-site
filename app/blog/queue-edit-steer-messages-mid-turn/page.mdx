# Line up your next moves while an agent is still working

**Who it's for:** anyone driving a long-running agent turn in the Omnigent web UI who wants to line up follow-ups without breaking the agent's flow.

## The problem it solves

You often know your next instruction before the agent finishes the current one — but sending it used to POST immediately, so a follow-up typed in a hurry was already committed with no way to take it back. Now that message is held in a **client-side queue** before it's sent: you keep steering the run without interrupting it, and you stay in control of exactly what reaches the agent and when. That's the orchestration layer doing its job — the same queue works whether the session runs an SDK or a native terminal harness.

## How to use

While the agent is busy, type a follow-up and press send — it lands in a docked strip above the composer marked **Queued** instead of interrupting the turn. Queued messages drain automatically, one at a time in order, as soon as the session goes idle. Each row has inline actions:

- **Steer** — send the message now, mid-turn, instead of waiting for the queue to flush.
- **Edit** — pull the message back into the composer to revise it.
- **Delete** — drop the message from the queue.
- **Reorder** — drag rows by the grip handle to change the order they'll send.

The queue is per-session and survives a hard refresh, and it flushes in the background even when you've navigated to another session.

See the [Web UI docs](/docs/interact/web-ui#message-queue-and-steering) for details.
