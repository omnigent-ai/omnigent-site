# Slack

The `@omnigent` Slack bot lets you talk to your agent from Slack, connecting to the same
persistent, shareable session as the terminal, web UI, mobile, and desktop interfaces.

The bot is managed with the `omni` CLI, under `omni integration slack`.

## Run the bot

```bash
omni integration slack              # run in the foreground (Ctrl-C to stop)
omni integration slack --background # run in the background (detached)
omni integration slack status       # is the background bot running?
omni integration slack stop         # stop the background bot
omni integration slack logs         # print the background bot's log path
omni integration slack logs -f      # follow the log (like tail -f)
```

A bare `omni integration slack` runs the socket-mode bot in the **foreground** — press
Ctrl-C to stop it.

## Run in the background

Pass `--background` to spawn the bot as a detached daemon that returns immediately:

```bash
omni integration slack --background
```

Running `omni integration slack --background` again while the daemon is already up is a
no-op that reports the existing process, so it's safe to re-run. Use the `status`, `stop`,
and `logs` subcommands to manage that background daemon.

## Configuration

All configuration — the two Slack tokens, `OMNIGENT_SERVER_URL`, and the optional
`OMNIGENT_DEVICE_CLIENT_SECRET` / `OMNIGENT_SLACK_TOKEN_ENCRYPTION_KEY` — comes from the
environment and the integration's `.env` file.
