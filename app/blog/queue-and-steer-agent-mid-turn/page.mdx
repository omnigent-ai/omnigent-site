# Line up your next messages and redirect an agent while it's still working

_For anyone pairing with an agent in the Omnigent web UI who thinks faster than the agent works._

## The problem it solves

When an agent is mid-turn, your next thought used to have nowhere to go — you either interrupted the run or sat on your hands until it finished. Now you can type follow-ups as they occur to you: each one is held in a client-side queue and drains automatically, one at a time in order, the moment the session goes idle. And when a thought can't wait, **steer** sends it into the running turn right now — on SDK harnesses the server live-injects it, so the agent folds it into the work in flight rather than after. Because the queue is per-session and client-side, you can line messages up in one session and switch to another; each one still lands in the session it was composed for, even carrying attachments.

## How to use

There's no CLI for this — it lives in the composer. While a turn is running, type a message and press Send: instead of interrupting, it docks in a strip above the composer. Each queued row carries inline actions:

- **Steer** — send it now, mid-turn, instead of waiting for the idle flush.
- **Edit** — pull it back into the composer to revise, then re-send.
- **Reorder** — drag rows to change the order they'll flush in.
- **Delete** — drop it from the queue.

Otherwise, the queue drains on its own — one message per idle, in order — including in the background while you're viewing another session.

See [Message queue and steering](/docs/interact/web-ui#message-queue-and-steering) in the docs.
