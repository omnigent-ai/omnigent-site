If you drive long agent runs in the web UI, you can now line up your next few
messages instead of waiting for each turn to finish. Type a follow-up while the
agent is still working and it is held in a client-side queue rather than
interrupting the running turn.

Before this, a message typed mid-turn either had to wait in your head or risk
landing in the wrong place. Now queued messages show in a docked strip above the
composer and drain automatically, one at a time and in order, the moment the
session goes idle. Each queued row also has inline actions, so a plan you lined
up early is not locked in: you can edit a message to revise it, drag to reorder
the queue, delete one you no longer want, or steer a message to send it now
instead of waiting for the queue to flush. Steer does not cancel the turn; the
agent folds the message into its current work at the next natural breakpoint,
and it works with any agent you run in Omnigent.

There is nothing to enable. While a turn is running, type into the composer and
press send: the message drops into the strip above it. Hover a queued row to
edit, delete, or steer it, and drag rows to change their order. See the
[Web UI guide](/docs/interact/web-ui#message-queue-and-steering) for the full
list of queued-row actions.
